# Expiry Convergence Strategy (到期收敛策略)

## 一句话解释

在市场结算前 24-72 小时，找到**结果已经可以验证但价格还没收敛到 100%** 的市场，买入等兑付。

---

## 为什么这个策略能赚钱？

Polymarket 的每个市场最终只有两种结局：YES 兑付 $1.00，或 NO 兑付 $1.00。

如果一个市场 YES 的结果已经"板上钉钉"了（比如股票已经涨到目标价之上），但 YES 的价格还在 $0.95 而不是 $1.00，那你花 $0.95 买入，等结算拿 $1.00，赚 $0.05/share。

**价格没到 $1.00 的原因**：
- 资金有时间成本（锁几小时到几天才能拿回来，年化不划算）
- 市场有 spread（买入价 > 显示概率）
- 有人还没更新认知（信息传导延迟）
- 极小概率的黑天鹅（结算规则歧义、平台风险等）

我们赚的就是**最后这段收敛的钱**。资金量小的时候，锁定几小时的时间成本可以忽略不计。

---

## 具体怎么操作

### Step 1: 扫描 (find_opportunities)

用 `find_opportunities` 工具扫描即将到期的高概率市场：

```
find_opportunities(
  min_probability = 0.90,    # 至少 90% 概率
  max_probability = 0.95,    # 不超过 95%（太高没利润）
  max_hours = 48,            # 48 小时内结算
  binary_only = true         # 只看 Yes/No 二元市场
)
```

返回一堆候选市场，每个带有：
- `probability`: Gamma API 估计的概率（参考值）
- `real_ask_price`: 实际买入价（你真正要付的钱）
- `end_date`: 交易截止时间
- `token_id`: 下单用的标识符

**利润 = 1.00 - real_ask_price**。如果 ask 是 $0.95，利润就是 5%。

### Step 2: 验证 (Research)

这是整个策略**最关键**的一步。不是看到 95% 就买，而是要**独立验证结果是否已确定**。

不同类型市场的验证方法：

| 市场类型 | 验证方法 | 示例 |
|----------|----------|------|
| 股票收盘价 | 查当前股价 vs 目标价，算缓冲 | META 现在 $643，目标 $630，缓冲 $13 |
| 天气 | 查 AccuWeather/NWS 预报 | NYC 预报最高 40F，阈值 46F，差 6F |
| 体育奖项 | 查投票结果/权威预测 | NFL ROTY 赔率已锁定某球员 |
| 财报 | 查已发布的财报数字 vs 市场阈值 | EPS $0.74 > 共识 $0.72 |
| 政治/法律 | 查官方公告、新闻源 | 超级碗赢家已定 |

**验证的核心问题**："从现在到结算，这个结果还有可能翻转吗？"

- 股价差 $13 但还有 4 小时交易 -> 翻转概率低但不是 0
- 天气预报差 6F 且是 24 小时后 -> 几乎不可能翻转
- 比特币差 $2000 但还有 12 小时 -> 完全可能翻转（我们亏过的教训）

### Step 3: 下单 (place_order)

验证通过后下单：

```
place_order(
  token_id = "xxx",
  side = "BUY",
  size = 20,          # 20 shares
  outcome = "YES",    # 或 "NO"
  event_slug = "meta-close-above-on-february-19-2026"
)
```

**下单后必须检查 `actual_avg_price`**。因为 order book 深度有限，实际成交价可能比显示的 ask 高。如果 >= $0.99，利润基本没了。

### Step 4: 等待结算 (Wait)

市场到期后，Polymarket 管理员确认结果，winning side 兑付 $1.00/share。我们的 `session_start.py` hook 自动检测已结算的仓位并 redeem 到 USDC。

## 策略的数学本质

假设你以 $P 的价格买入一个你有 $q$ 信心会赢的市场：

```
期望收益 = q * (1 - P) - (1 - q) * P
         = q - P
```

所以只要 **你的信心 q > 市场价格 P**，期望就是正的。

关键洞察：我们不需要 q 比 P 高很多。市场定价 $0.95，如果我们验证后认为 q = 0.99，期望收益 = 0.99 - 0.95 = 0.04/share (4%)。

**我们的 edge 不是"预测未来"，而是"验证现在"。** 市场价格反映的是所有参与者的平均认知，但很多参与者不会在最后几小时去核实最新的天气预报或股价。我们做了这一步，所以我们的 q 比市场的 P 更准。

## 风险管理

| 参数 | 值 | 作用 |
|------|-----|------|
| MAX_BUY_PRICE | $0.99 | 买入价 >= $0.99 自动拒绝（利润太薄） |
| MAX_POSITION_PCT | 20% | 单个市场最多投入余额的 20% |
| STOP_LOSS_DROP_PCT | 20% | 价格跌超 20% 自动卖出止损 |
| max_hours | 48h | 只看 48 小时内到期的市场（减少锁仓时间） |

**止损是双刃剑**：BTC 那次就是止损触发了（从 $0.91 跌到自动卖在 $0.49），避免了更大损失（最终归零）。但如果是假跌又回来的情况，止损会锁定亏损。对于到期收敛策略，止损主要防的是"验证错了"的情况。

## Agent 的工作流程

```
每 2-4 小时自动唤醒（scheduler）
        |
        v
[session_start hook]
  - 自动 redeem 已结算的仓位
  - 同步链上余额和持仓
  - 注入当前状态到 prompt
        |
        v
[Agent 扫描]
  find_opportunities(min=0.90, max=0.95, max_hours=48)
        |
        v
[Agent 验证]
  对每个候选：WebSearch 查新闻、查股价、查天气...
  "这个结果确定了吗？从现在到结算还能翻转吗？"
        |
    确定 / 不确定
     |         \
     v          v
  place_order   HOLD（写 trading_log 记录原因）
     |
     v
[Agent 休眠]
  hibernate(N, "D-mail: 下次醒来检查什么")
        |
        v
  等待 N 小时后重复...
```

## 策略局限性

1. **利润率低**: 单笔 3-5%，需要高频周转才能累积
2. **依赖市场供给**: 不是随时都有符合条件的市场，经常空扫（agent 连续 HOLD 多次很正常）
3. **验证不是 100% 可靠**: 天气预报可能错、股价可能最后一刻暴跌
4. **资金效率**: 钱锁在仓位里等结算，期间不能用于其他交易
5. **规模上限**: 低流动性市场吃不了大单，$100 级别刚好，$10K 级别就会推动价格

## 适合什么人/什么资金量？

- 小资金 ($100-$1000)：理想。时间成本可忽略，流动性够用
- 中资金 ($1K-$10K)：可以但需要分散到更多市场
- 大资金 ($10K+)：不太适合，利润率太低且流动性不足

---

## 与其他策略的关系

到期收敛是我们的**基础策略**（当前唯一在跑的）。未来可能叠加：

- **关联市场套利**: 发现定价不一致的关联市场（见 `cross-market-arbitrage.md`）
- **事件驱动**: 在事件发生瞬间（如财报发布）快速交易，利用信息传导延迟
- **做市**: 双边挂单赚 spread（需要更大资金和不同架构）
